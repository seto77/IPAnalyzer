# Find Parameter (Brute force)

Режим полного перебора варьирует каждый параметр исчерпывающим образом (поиск по сетке), чтобы найти значения, наилучшим образом удовлетворяющие условиям:

- острые пики (то есть малая FWHM) и
- малое отклонение значений d.

Он эффективен для неполных колец или зашумлённых данных, где геометрической оптимизации трудно сойтись. Обзор инструмента см. в разделе [Инструменты](3-tools.md), а основной порядок действий — в разделе [Процедуры](4-procedures.md).

## Steps

1. Загрузите данные изображения и параметры, затем нажмите **Find Parameter (brute force)**.

    ![Launching Find Parameter (brute force)](https://user-images.githubusercontent.com/44538886/180194930-bcefbf4f-7b61-4068-96d1-18bb29109411.png)
    ![Brute-force calibration window](../assets/cap-en-auto/FormFindParameterBruteForce.png)

2. Выберите **standard material**. Если его нет в списке, выберите **Others** и введите его вручную.
3. Отобразите профиль с помощью **Get profile**.
4. Дважды щёлкните по любой пиковой линии, которую вы не хотите использовать для оптимизации, чтобы исключить её.
5. Задайте параметры и нажмите **Optimize**, чтобы запустить оптимизацию.

## Options

![Option settings](https://user-images.githubusercontent.com/44538886/180196763-e707ea11-5594-4cfe-876b-6d7d28b775e8.png)

### Fitting range

Задаёт угловой диапазон, по которому выполняется аппроксимация каждого пика.

### Number of repetitions

Задаёт число циклов оптимизации.

### Per-parameter options

Оптимизация выполняется по отмеченным элементам. Каждый параметр перебирается в указанном диапазоне (**Search range**) с указанным шагом (**Initial step**).

Например, если длина камеры начинается со 100 мм при Initial step 0,05 мм и Search range 4, то перебираются 9 (= Search range × 2 + 1) значений длины камеры: 99,80, 99,85, 99,90, 99,95, 100,00, 100,05, 100,10, 100,15, 100,20 мм.

Когда числа испытаний для параметров равны N1, N2, N3, …, общее число испытаний за один цикл составляет N1 × N2 × N3 × …. В приведённом выше примере с четырьмя параметрами при Search range 4, 4, 3 и 6 это даёт (4×2+1) × (4×2+1) × (3×2+1) × (6×2+1) = 7371 испытание. Иными словами, перебираются 7371 набор параметров, и выбирается комбинация с самыми острыми пиками и наименьшим отклонением значений d.

!!! warning
    Увеличение **Search range** резко увеличивает число испытаний и удлиняет каждый цикл. Используйте его с осторожностью.

## Status screen

![Status screen](https://user-images.githubusercontent.com/44538886/180197340-efeb9fd5-4e38-4845-939e-1352903502d0.png)

Во время калибровки отображается экран состояния, подобный показанному выше.

- **Cycle**: номер текущего цикла. Когда цикл завершается, найденный в нём наилучший набор параметров становится начальной точкой следующего цикла. Если наилучшие параметры совпадают с параметрами предыдущего цикла, шаг умножается на 0,8 для следующего цикла.
- **Current best values**: наилучшие параметры на данный момент.
- **Current steps**: шаг каждого параметра на данный момент.
- **No1, No2, …**: пять лучших значений оценки в цикле. Сначала они быстро убывают и сходятся по мере приближения к оптимуму.
